**优先级：** P0
**模块：** 全局
**状态：** Completed

### F-002 · 账号登录与类型识别

**优先级：** P0
**模块：** 全局
**状态：** Completed

**功能描述**
用户通过统一密码登录，默认以 Member 身份进入系统。登录后可通过 Sidebar 用户菜单随时切换为 Admin 身份，无需重新登录。

**登录流程**
1. 输入任意邮箱（格式校验仅做基础验证）
2. 输入统一密码 `123`
3. 点击登录 → 默认以 **Member** 身份进入首页

**账号切换**
- 点击 Sidebar 底部用户头像（samhar / admin）→ 弹出下拉菜单
- 选择「切换为 Admin」/「切换为 Member」→ 即时切换，页面刷新至对应首页
- 切换时清空当前项目/任务状态，返回首页

**权限**
- Member：渲染 HubAgent 首页，无组织管理导航
- Admin：渲染 OrgAgent 首页（数据看板 + OrgAgent 对话区），显示「组织管理」导航组
- Member 访问 Admin 专属路由（`/settings/*`、`/org/*`）→ 302 重定向首页

**上下文**
- 登录后，系统初始化该用户的 HubAgent 实例（若首次登录）
- Admin 登录后，系统检查 OrgAgent 实例是否存在（全组织唯一）

**验收标准**
- [x] 密码统一为 `123`，任意邮箱均可登录
- [x] 登录默认进入 Member 首页
- [x] Sidebar 用户菜单支持 Member / Admin 切换
- [x] 切换后返回首页，清空项目/任务状态
- [x] Member 访问 Admin 路由被重定向
- [ ] 首次登录时 HubAgent 完成初始化

### F-003 · 首页数据看板 — Member

**优先级：** P0
**模块：** 首页 `/dashboard`
**状态：** InProgress

**功能描述**
Member 首页左列的数据看板，展示个人维度的跨项目数据概览。右列为 HubAgent 对话区（见 F-004）。

**布局**
```
┌──────────────────────┬──────────────────────┐
│  数据看板（左 60%）   │  HubAgent 对话区（右 40%）│
└──────────────────────┴──────────────────────┘
```

**KPI 卡片（3 张）**

| 卡片 | 数据范围 | 点击行为 |
|------|---------|---------|
| 本月 Token 消耗 | 该 Member 参与的所有项目 | 展开消耗明细 |
| 活跃项目 | 有进行中任务的项目数 | 跳转项目列表 |
| 任务成功率 | 该 Member 创建并完成的任务（近 7 日）| 展开任务统计 |

**中部：项目消耗概览表**
- 列：项目名 / 本月消耗 / 占比进度条 / 趋势
- 仅显示该 Member 参与的项目（已加入的项目）
- 按消耗量降序，点击行跳转项目页

**底部：近 7 日消耗折线图**

**项目上下文记忆**
在消耗折线图下方，展示该 Member 所有项目的上下文记忆文档列表。每个项目对应一份 `context.md` 文档（即项目级上下文，由该项目下所有任务的 context.md 自动汇总生成，WorkAgent 实时维护）。

```
📄 供应链金融风控 · context.md
📄 智能客服知识中台 · context.md
📄 市场竞品分析 · context.md
```

- 每个文档条目显示项目名称 + "context.md"
- 点击文档条目 → 自动 `@项目名称 · context.md` 到右侧 HubAgent 输入框，作为 HubAgent 的上下文引用
- 被引用的 context.md 会随用户消息一起发送给 HubAgent，HubAgent 可基于该项目的完整上下文进行跨项目分析

**与 HubAgent 的联动**
| 操作 | 效果 |
|------|------|
| 点击项目 context.md | HubAgent 输入框自动填充 `@项目名 · context.md` 标签 |
| 发送消息 | HubAgent 接收消息时自动加载该项目 context.md 的上下文 |
| 多选 | 支持同时引用多个项目的 context.md |

**上下文范围**
- 数据严格限定在该 Member 参与的范围内
- 不显示未参与项目的任何数据

**权限**
- 仅 Member 账号渲染此看板
- Admin 账号渲染 F-008 的全组织看板

**验收标准**
- [ ] KPI 卡片数据仅反映当前 Member 的个人维度
- [ ] 项目消耗表仅显示该 Member 自己的项目
- [ ] 数据加载中有 skeleton 占位
- [ ] 项目 context.md 列表正确展示所有项目
- [ ] 点击 context.md 自动 `@` 引用到 HubAgent 输入框
- [ ] HubAgent 能正确加载引用的 context.md 上下文

### F-004 · HubAgent 对话区

**优先级：** P0
**模块：** HubAgent（首页 / Skills / Tools 右列）
**状态：** InProgress

**功能描述**
HubAgent 对话区是 Member 与 HubAgent 交互的统一界面，在首页、Skills 页、Tools 页的右列复用同一组件。页面切换时对话历史保留，上下文随场景自动切换。

**上下文切换规则**

| 所在页面 | HubAgent 上下文模式 | 能力范围 |
|---------|-----------------|---------|
| 首页 | 全局模式 | 跨项目问答、项目创建 |
| Skills 页 | Skill 管理模式 | Skill 创建引导、使用统计查询 |
| Tools 页 | Tools 管理模式 | Tool 连接配置、权限设置、费用查询 |

页面切换时，HubAgent 顶部显示短暂的上下文切换提示（「已切换到 Skill 管理模式」），2 秒后消失。

**记忆归属**
- 对话历史存入该 Member 的 HubAgent 记忆（个人私有）
- HubAgent 记忆不跨 Member 共享
- HubAgent 不可访问其他 Member 的记忆，不可访问 WorkAgent 的项目记忆

**权限**
- 仅 Member 账号有 HubAgent
- Admin 账号的首页为 OrgAgent，无 HubAgent

**界面结构**
```
┌──────────────────────────────────────┐
│ HubAgent                    ● 在线   │
├──────────────────────────────────────┤
│  对话区（流式输出，消息历史）          │
│  空状态：展示快捷引导问题             │
├──────────────────────────────────────┤
│ 输入消息...                     [→]  │
└──────────────────────────────────────┘
```

**验收标准**
- [ ] 流式输出正常渲染，支持 Markdown
- [ ] 页面切换时对话历史不丢失
- [ ] 上下文切换提示短暂显示后消失
- [ ] `Enter` 发送，`Shift+Enter` 换行

### F-005 · 新建任务页（独立任务，无需项目）

**优先级：** P1
**模块：** WorkAgent / 独立任务
**状态：** Completed

**功能描述**
Member 可以不依赖任何项目，直接通过 Sidebar「新建对话」创建独立任务。独立任务同样由 WorkAgent 驱动、Sub-Agent 执行，与项目内任务的执行机制完全一致，区别仅在于它不归属于任何项目。

适用场景：临时性分析、个人头脑风暴、不成体系的单次任务，无需先创建项目再进入再创建任务。

**路由：** `/task/new` 或 `/task/:taskId`

**Sidebar 交互**

Sidebar 中「新建对话」是一个独立的可折叠分组，位于顶部导航（首页/Skills/Tools）与「我的项目」之间：

```
∨ 新建对话
─────────────────────
我的项目              [+]
▼ 供应链金融风控      ●
    ...
```

- 点击区域进入空白的新任务页
- 历史独立任务以对话标题展示，点击可重新进入
- 分组默认展开，可折叠

**与项目任务的区别**

| 维度 | 独立任务（新建对话）| 项目内任务 |
|------|----------------|----------|
| 创建权限 | 所有 Member 均可自主创建 | 仅项目创建者可创建 |
| 归属 | 不属于任何项目 | 归属某个项目 |
| 驱动 Agent | WorkAgent（临时实例）| WorkAgent（该用户的项目实例）|
| 上下文记忆 | 仅本次任务内（无跨任务记忆）| 该用户个人项目 WorkAgent 的历史记忆 |
| 产出写入图谱 | 不写入任何项目图谱 | 写入该用户个人项目知识图谱 |
| 成员协作 | 无协作，仅创建者可访问 | 仅创建者可访问 |
| 右侧面板 Tab | 任务树 / 图谱（无项目树） | 任务树 / 项目树 / 图谱 |

**执行机制**

独立任务页与项目内任务页的界面和执行逻辑完全相同（WorkAgent + Sub-Agent 动态编排），不需要独立开发执行逻辑，复用任务页组件即可。

差异仅在于：
1. WorkAgent 为临时实例，无项目历史上下文，任务结束后上下文不持久化到项目层
2. 独立任务的右侧面板不显示「项目树」Tab（因为不属于任何项目）

**记忆归属**
- 独立任务的执行过程归创建者私有
- 不写入任何 WorkAgent 项目记忆
- 对话历史和产出文件归创建者个人，可在任务页查看

**权限**
- 所有 Member 均可创建独立任务
- Admin 暂不支持（Admin 无 WorkAgent 入口）
- 独立任务对其他 Member 不可见

**验收标准**
- [x] Sidebar「新建对话」分组位置正确（顶部导航与「我的项目」之间）
- [x] 点击进入空白新任务页，WorkAgent 就绪
- [x] 历史独立任务按最近使用时间倒序展示在分组下
- [x] 独立任务页复用项目内任务页的执行流组件
- [ ] 任务产出不写入任何项目的知识图谱
- [ ] 其他 Member 无法访问该独立任务

### F-014 · WorkAgent — 项目上下文记忆

**优先级：** P0
**模块：** WorkAgent
**状态：** Planned

**功能描述**
WorkAgent 持久存储该用户在此项目的完整上下文记忆，以**项目级 context.md 文档** + **知识图谱**的形式存在。项目级 context.md 由所有任务的 context.md 自动汇总生成，包含历史任务产出摘要、项目目标等文本内容；知识图谱存储抽取的实体/关系。每个用户的项目记忆独立，不与其他用户共享。在新任务中 WorkAgent 自动引用该用户的历史上下文。

**记忆内容**

| 记忆类型 | 来源 | 更新时机 | 可见范围 |
|---------|------|---------|---------|
| 任务产出摘要 | 该用户的任务完成后写入 | 每次任务完成 | 仅该用户 |
| 知识图谱 | 该用户任务产出中抽取的实体/关系 | 每次任务完成 | 仅该用户 |
| 项目目标/约束 | 用户手动录入或 WorkAgent 推断 | 随时更新 | 仅该用户 |
| 用户画像 | 基于该用户历史任务学习 | 持续积累 | 仅该用户 |

**记忆流向**
```
任务记忆（私有：任务执行过程）
  ↓ 任务完成后，仅产出部分写入
WorkAgent 个人项目记忆（用户私有：产出 + 图谱）
  ↓ 定期 Summary
HubAgent 记忆（该用户）
```

不写入项目记忆的内容：Sub-Agent 执行过程、中间步骤、失败记录、任务创建者的个人对话。

**上下文引用示例**
```
用户：分析今年的供应商数据

[W] 基于你上次（8月）的供应商评估框架：
    · 上次评估了 47 家，其中 3 家高风险
    · 天合光能数据有缺口，本次需要特别处理
    本次方案：沿用上次模型，更新 Q4 数据
    [确认方案]  [修改方案]
```

**权限**
- 项目记忆归用户个人所有（per-user per-project），不与其他用户共享
- 用户离职后，其个人项目记忆可被 Admin 查看状态，但内容不可见
- Admin 不可查看任何用户的项目记忆内容

**验收标准**
- [ ] 任务完成后 WorkAgent 自动更新该用户的项目记忆
- [ ] 新任务发起时 WorkAgent 自动引用该用户的历史产出
- [ ] 不同用户的项目记忆互相隔离，不可访问
- [ ] 用户可在 WorkAgent 对话中查看和管理自己的项目记忆

### F-031 · HubAgent — OrgAgent Summary 上报

**优先级：** P2
**模块：** HubAgent / OrgAgent
**状态：** Planned

**功能描述**
项目级上下文记忆 MD 文档自动同步到首页，供 Member 和 Admin 通过各自 Agent 引用分析。

**数据流三层模型**

```
任务页 → 任务级上下文 MD → 项目页汇总
                                    ↓
项目页 → 项目级上下文 MD + 知识图谱
            ↓                              ↓
    Member 首页                      Admin 首页
    HubAgent 可 @ 引用               OrgAgent 可 @ 引用
    （仅自己的项目）                  （所有用户的项目）
```

**各层说明**

| 层级 | 文档 | 生成方式 | 同步目标 |
|------|------|---------|---------|
| 任务级 | 任务上下文 MD | 每次任务完成后 WorkAgent 写入 | 同步到项目页 |
| 项目级 | 项目上下文 MD + 知识图谱 | 汇总该项目下所有任务文档 | 同步到首页 |
| 用户级 | — | Member 仅看到自己的项目文档 | HubAgent |
| 组织级 | — | Admin 看到所有用户的项目文档 | OrgAgent |

**权限**
- Member：仅能在 HubAgent 引用自己项目的上下文文档
- Admin：能在 OrgAgent 引用全组织所有项目的上下文文档
- 个人对话内容、任务执行过程不写入任何层级文档

**验收标准**
- [ ] 任务完成后任务级文档同步到项目页
- [ ] 项目页正确汇总生成项目级上下文文档
- [ ] Member 首页正确展示自己项目的上下文文档
- [ ] Admin 首页正确展示全组织所有项目的上下文文档
- [ ] 个人对话内容不出现在任何层级文档中
