## 一、产品概述

### 1.1 定位与愿景

AgentOS 是一个面向 B 端团队的 AI 原生项目协同平台。它将传统的「人管项目」升级为「人 + AI Agent 协同驱动项目」，每个项目由人类员工和数字员工（AI Agent）共同完成。平台提供完整的项目生命周期管理、Agent 技能编排、以及人机自然语言交互能力。

**核心转变**：从「人管理任务」到「人指挥 Agent，Agent 执行任务」。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **项目** | 业务场景载体，如「供应链金融风控平台」「智能客服升级」|
| **任务** | 项目内的具体工作单元，由 WorkAgent 编排 Sub-Agent 执行 |
| **数字员工（Agent）** | 具备特定领域能力的 AI 代理，如财报分析 Agent、风控评估 Agent |
| **Skills** | Agent 的能力单元，可被挂载到不同 Agent 上复用，如「图表生成」「数据清洗」|
| **Tools** | 外部系统连接，如企查查 MCP、飞书 API、数据库等 |
| **HubAgent** | Member 的个人全局助手，提供跨项目问答、审批处理、工具管理 |
| **WorkAgent** | 项目级共享实例，负责理解意图、拆解任务、编排 Sub-Agent |
| **Sub-Agent** | 任务执行中的动态 Agent，调用 Skill/Tool 完成原子操作，任务结束后消亡 |
| **OrgAgent** | Admin 的组织级助手，提供全组织效能分析与战略洞察 |

### 1.3 产品目标

1. **降低 AI 使用门槛**：通过自然语言对话即可调度 Agent 和 Skill 完成任务
2. **提升项目透明度**：任务树实时展示 Sub-Agent 执行状态，审批流程清晰可见
3. **人机无缝协作**：人类负责决策和审核，Agent 负责执行和产出，Skill 提供原子能力
4. **知识自动沉淀**：任务产出自动写入项目记忆和知识图谱，供后续任务引用

## 二、用户与角色

### 2.1 账号类型

| 账号 | 定位 | 首页 Agent | 可见导航 |
|------|------|-----------|---------|
| **Member** | 任务执行者，在项目中完成具体工作 | HubAgent | 首页、Skills、Tools、我的项目 |
| **Admin** | 团队负责人，关注组织效能与资源分配 | OrgAgent | 首页、Skills、Tools、我的项目、组织管理 |

> Admin 不是 IT 系统管理员，而是业务层的团队负责人（如部门 Leader、项目总监）。系统级配置由 IT 管理员通过独立的管理后台处理。

### 2.2 项目内角色

| 项目角色 | 产生方式 | 权限 |
|---------|---------|------|
| **项目负责人** | 项目创建者自动成为负责人 | 创建任务、派发任务、审批 Sub-Agent 操作、成员管理 |
| **项目成员** | 被负责人派发任务后自动加入项目 | 执行被派发的任务、查看项目共享数据 |

> 项目角色与账号类型独立。Admin 也可以是某个项目的负责人；普通 Member 在项目 A 是成员，也可以是项目 B 的负责人。

### 2.3 角色权限矩阵

| 操作 | Member（普通成员）| Member（项目负责人）| Admin |
|------|----------------|------------------|-------|
| 创建项目 | ✓ | ✓ | ✓ |
| 创建任务（项目内）| ✗ | ✓ | ✓（若负责人）|
| 派发任务 | ✗ | ✓ | ✓（若负责人）|
| 审批 Sub-Agent | ✗ | ✓ | ✓（若负责人）|
| 查看全部任务 | ✗ | ✓ | ✓（若负责人）|
| 成员管理 | ✗ | ✓ | ✓（若负责人）|
| 访问组织管理 | ✗ | ✗ | ✓ |
| 查看 OrgAgent | ✗ | ✗ | ✓ |

## 四、Agent 体系

### 4.1 四种 Agent 定位

```
┌────────────────────────────────────────────────────────────────┐
│  Admin 首页                                                     │
│  OrgAgent — 全组织视野，战略洞察，效能管理                       │
├────────────────────────────────────────────────────────────────┤
│  Member 首页 / 新建对话 / Skills 页 / Tools 页                   │
│  HubAgent — 个人全局视野，自由对话，审批处理，工具管理            │
├───────────────────────┬────────────────────────────────────────┤
│  项目 A               │  项目 B                                 │
│  WorkAgent（独立实例） │  WorkAgent（独立实例）                   │
│  项目级上下文记忆      │  项目间互相隔离                          │
├───────────────────────┴────────────────────────────────────────┤
│  任务执行空间（任务页）                                           │
│  WorkAgent（编排者）+ Sub-Agent × N（动态执行者）                │
└────────────────────────────────────────────────────────────────┘
```

| Agent | 场景 | 实例数量 | 生命周期 | 核心能力 |
|-------|------|---------|---------|---------|
| **HubAgent** | 首页、新建对话、Skills、Tools | 每个 Member 一个 | 账号存续期间持久 | 跨项目问答、审批汇集、Skill/Tool 管理 |
| **WorkAgent** | 项目页、任务页 | 每个项目一个 | 项目存续期间持久 | 任务拆解、Sub-Agent 编排、项目记忆管理 |
| **Sub-Agent** | 任务执行中 | 动态，数量不固定 | 单次任务，完成后消亡 | 调用 Skill/Tool 完成原子操作 |
| **OrgAgent** | Admin 首页 | 全组织一个 | 持久 | 全组织效能分析、战略建议、资源预警 |

### 4.2 Agent 协作关系

**任务执行链路**：
```
Member 输入指令
  ↓
WorkAgent 理解意图 + 引用项目记忆
  ↓
WorkAgent 生成执行方案（Sub-Agent 规划、Skill/Tool 选择、消耗预估）
  ↓
Member 确认方案
  ↓
WorkAgent 动态创建 Sub-Agent 并编排执行
  ↓
Sub-Agent 调用 Skill / Tool 完成原子操作
  ↓
Sub-Agent 汇报执行结果（折叠卡片）
  ↓
WorkAgent 汇总产出 → 写入项目记忆
```

**审批链路**：
```
Sub-Agent 触发审批策略（费用超阈值）
  ↓
WorkAgent 暂停 Sub-Agent
  ↓
推送审批请求至项目负责人 HubAgent
  ↓
负责人在 HubAgent 处理（批准 / 拒绝）
  ↓
HubAgent 回传结果至 WorkAgent
  ↓
Sub-Agent 恢复执行（批准）或终止该分支（拒绝）
```

### 4.3 Agent 生命周期

| Agent | 创建时机 | 销毁时机 |
|-------|---------|---------|
| HubAgent | Member 首次登录时初始化 | 账号注销 |
| WorkAgent | 项目创建时同步初始化 | 项目归档 |
| Sub-Agent | WorkAgent 编排任务时动态创建 | 任务完成或失败后消亡 |
| OrgAgent | 组织创建时初始化 | 组织解散 |

## 五、上下文与记忆模型

### 5.1 四层记忆结构

```
OrgAgent 记忆（全组织层）
  ├── 来源：所有 HubAgent 定期上报的 Summary（脱敏）
  ├── 来源：公司战略层数据（Admin 手动输入或导入）
  ├── 视野：全组织效能、资源分配、趋势分析
  └── 归属：组织（Admin 可访问）

HubAgent 记忆（个人层）
  ├── 来源：该 Member 的所有对话历史（含自由对话）
  ├── 来源：该 Member 参与的所有项目的个人操作记录
  ├── 来源：该 Member 处理的审批记录
  ├── 视野：这个人的全局工作状态
  └── 归属：该 Member 个人私有

WorkAgent 记忆（项目层）
  ├── 来源：项目内所有已完成任务的产出摘要
  ├── 来源：任务执行后写入的知识图谱（实体、关系）
  ├── 来源：项目目标、约束、已识别风险
  ├── 视野：这个项目的全部知识积累
  └── 归属：项目（项目内所有成员共享）

任务记忆（任务层）
  ├── 来源：WorkAgent 在此任务中的编排过程
  ├── 来源：Sub-Agent 的执行过程（调用链、中间结果）
  ├── 视野：这一个任务的完整执行过程
  └── 归属：任务执行者私有
```

### 5.2 记忆流向规则

```
任务记忆
  ↓ 任务完成后，仅产出部分写入
WorkAgent 项目记忆
  ├── 写入内容：任务产出物摘要、新增图谱实体/关系、关键结论
  └── 不写入：Sub-Agent 执行过程、中间步骤、失败记录

WorkAgent 项目记忆
  ↓ 定期 Summary
HubAgent 记忆（该项目负责人）
  └── 写入内容：项目进展摘要、Token 消耗、任务完成情况

HubAgent 记忆
  ↓ 定期 Summary（脱敏）
OrgAgent 记忆
  ├── 写入内容：Token 消耗统计、任务成功率、效能指标
  └── 不写入：个人对话内容、任务具体细节、私人信息
```

### 5.3 记忆隔离边界

**HubAgent 记忆对 OrgAgent 的可见性**：

| 数据类型 | OrgAgent 可见 | 说明 |
|---------|-------------|------|
| Token 消耗统计 | ✓ | 脱敏聚合数据 |
| 任务成功率 | ✓ | 统计指标 |
| 项目进展状态 | ✓ | 仅状态，无内容 |
| 个人对话内容 | ✗ | 绝对私有 |
| 自由对话内容 | ✗ | 绝对私有 |
| 审批处理记录 | ✓ | 操作日志，无内容详情 |

**WorkAgent 项目记忆的访问权限**：

| 角色 | 可访问内容 |
|------|-----------|
| 项目负责人 | 全部项目记忆 |
| 项目成员 | 任务产出物、图谱（只读）|
| 非项目成员 | 无访问权限 |
| Admin | 项目存在性和状态，不可查看内容 |

**任务记忆的访问权限**：

| 角色 | 可访问内容 |
|------|-----------|
| 任务执行者 | 完整任务记忆（执行过程 + 产出）|
| 项目负责人 | 仅任务产出摘要（不含执行过程）|
| 其他项目成员 | 无访问权限 |

## 七、页面功能详述

### 7.1 首页（Dashboard）

**路由**：`/dashboard`

**布局**：左右分栏（左 60% 数据看板 / 右 40% Agent 对话区）

#### Member 视图

**左列 — 数据看板**：

| 模块 | 内容 | 点击行为 |
|------|------|---------|
| **KPI 卡片（4 张）** | 本月 Token 消耗 / 活跃项目 / 任务成功率 / 待处理审批 | 展开明细 / 跳转列表 / 展开统计 / 跳转 HubAgent |
| **项目消耗概览表** | 项目名 / 本月消耗 / 占比进度条 / 趋势 | 点击跳转项目页 |
| **近 7 日消耗折线图** | Token 消耗趋势 | — |

> 「待处理审批」仅在该 Member 是项目负责人且有待审批时显示 `●`，普通成员此卡片显示空态。

**右列 — HubAgent 对话区**：

- 顶部显示上下文模式（全局/Skill 管理/Tools 管理），切换时短暂提示 2 秒后消失
- 流式输出支持 Markdown
- 空状态展示快捷引导问题
- `Enter` 发送，`Shift+Enter` 换行

#### Admin 视图

**左列 — 组织看板**：

| 模块 | 内容 | 点击行为 |
|------|------|---------|
| **KPI 卡片（4 张）** | 全组织 Token 消耗 / 活跃团队 / 全局任务成功率 / 待处理预警 | 跳转用量管理 / 展开列表 / — / 跳转 OrgAgent |
| **组织健康摘要表** | 团队名 / 活跃成员 / 本月消耗 / 预算 / 状态 | 消耗 ≥ 80% 橙色高亮，点击行 OrgAgent 加载该团队上下文 |

**右列 — OrgAgent 对话区**：

- 首次进入自动展示本周组织摘要
- 战略层分析、跨团队对比、趋势预测
- 不可查询任何 Member 个人对话

### 7.2 项目页

**路由**：`/project/:projectId`

**布局**：左列（任务列表 + 项目信息）+ 右列（WorkAgent 对话区）

**页面顶部**：
```
[项目名称]
负责人：[姓名]  ·  成员：N 人  ·  活跃任务：N 个
[成员管理]  [项目设置]          ← 仅负责人可见
```

**Tab 导航**：`[任务]`（默认）`[文件]` `[数据]` `[图谱]`

**任务列表项**：

| 字段 | 说明 |
|------|------|
| 任务名称 | 点击进入任务页 |
| 状态 | ● 运行中 / ⏸ 等待审批 / ✓ 已完成 / ○ 未开始 / ✗ 失败 |
| 执行者 | 头像 + 姓名 |
| 最后更新 | 相对时间 |
| 操作 | `[进入]` |

**权限差异**：
- 普通成员：仅见自己被派发的任务
- 负责人：可见全部任务 + 「+ 新建任务」入口 + 成员管理 + 项目设置

### 7.3 任务页

**路由**：`/project/:projectId/task/:taskId`

**布局**：左侧（对话与执行流）+ 右侧（任务树面板，30%）

**任务页固定组成**：
```
1 个 Member（任务执行者）
+ WorkAgent（编排者）
+ N 个 Sub-Agent（动态执行者）
```

#### 消息类型规范

| 类型 | 对齐 | 标识 | 样式 |
|------|------|------|------|
| Member 指令 | 右对齐 | 头像 | 普通气泡 |
| WorkAgent 消息 | 左对齐 | `[W]` 橙红色 | 宽卡片 |
| Sub-Agent 汇报 | 缩进左对齐 | 分身名称 | 灰色折叠卡片，默认收起 |

**Sub-Agent 汇报折叠卡片**：
```
┌─ Sub-Agent [数据拉取]  ✓ 2.1s ──────────────────────┐
│  从飞书拉取 47 份财报文件                            │
│  格式：Excel 32 份，PDF 15 份                        │
│                                       [展开详情]     │
└──────────────────────────────────────────────────────┘
```

**任务完成汇总**（WorkAgent 生成）：
```
[W] WorkAgent  ·  任务完成

  执行摘要：处理财报 47 份 · 高风险供应商 3 家
  总耗时：2h 14min  ·  消耗：18.4K tokens

  产出物：
  [供应商风险评估报告 Q4.pdf]    [下载]  [预览]
  [高风险供应商列表.xlsx]        [下载]  [预览]
```

#### 审批中断卡片

```
[W] WorkAgent  ·  需要审批

┌─────────────────────────────────────────────────────┐
│  ⏸ Sub-Agent [风险评估] 已暂停                      │
│  需要调用企查查 API                                  │
│  查询：47 家供应商  ·  预估费用 $240                 │
│  已通知 samhar（项目负责人）                         │
│                         [催审批]  [跳过此步骤]       │
└─────────────────────────────────────────────────────┘
```

- **催审批**：重新发送通知，冷却时间 1 小时
- **跳过此步骤**：Sub-Agent 跳过审批操作继续执行，完成汇总标注「已跳过」

#### 任务树实时面板（右侧）

```
┌──────────────────────────────┐
│ WorkAgent          ● 编排中  │
├──────────────────────────────┤
│ ├─ ✓ [数据拉取]   2.1s  1.2K│
│ ├─ ⟳ [财报解析]   进行中 4.5K│
│ ├─ ○ [风险评估]   等待中      │
│ └─ ○ [报告生成]   等待中      │
│                              │
│ 消耗：5.7K / 预计 18K        │
│ ████░░░░░░░  31%             │
│                              │
│           [查看完整 Trace]   │
└──────────────────────────────┘
```

节点状态：`✓` 完成 / `⟳` 运行中 / `○` 等待 / `⏸` 审批暂停 / `✗` 失败

### 7.4 Skills 页

**路由**：`/skills`

**布局**：左列（Skill 列表）+ 右列（HubAgent Skill 管理模式）

**Skill 来源与可见性**：

| 类型 | 可见条件 |
|------|---------|
| 预设 Skill | 所有用户 |
| 自建 Skill（私有）| 仅创建者 |
| 自建 Skill（项目内共享）| 该项目成员 |
| 自建 Skill（全局）| 所有用户 |

**列表项字段**：Skill 名称 / 类型标签 / 可见范围标签 / 调用次数 / 成功率 / 操作（编辑/详情）

**Skill 创建流程**（HubAgent 引导）：
```
用户：我需要一个能自动生成财务摘要报告的 Skill

[H] 请确认以下配置：
    名称：财务摘要生成
    输入：财务数据文件（CSV / Excel）
    输出：结构化摘要报告（Markdown）
    执行逻辑：提取关键指标 → 计算同比环比 → 生成摘要
    可见范围：[私有 ▼]

    [确认创建]  [修改配置]
```

> 新建 Skill 默认私有，仅创建者可编辑，预设 Skill 只读。

### 7.5 Tools 页

**路由**：`/tools`

**布局**：左列（Tool 连接列表）+ 右列（HubAgent Tools 管理模式）

**Tool 分类**：MCP / OAuth / VPN / API Key

**列表项字段**：Tool 名称 / 类型标签 / 连接状态（● 已连接 / ○ 未连接）/ 本月调用次数 / 累计费用 / 操作（管理/断开/连接）

**费用阈值设置**：
```
用户：企查查单次调用超过 $50 需要我审批

[H] 已设置企查查 MCP 调用策略：
    预估费用 ≤ $50 → Sub-Agent 自动执行
    预估费用 > $50 → 暂停，发送审批请求到你的 HubAgent
```

> 此规则写入该 Member 在所有项目中的 Sub-Agent 审批策略，跨项目生效。

### 7.6 新建独立任务页

**路由**：`/task/new` 或 `/task/:taskId`

**入口**：Sidebar「新建对话」→ 进入空白新任务页

**与项目内任务的区别**：

| 维度 | 独立任务 | 项目内任务 |
|------|---------|-----------|
| 创建权限 | 所有 Member | 仅项目负责人 |
| 归属 | 不属于任何项目 | 归属某个项目 |
| WorkAgent 实例 | 临时实例 | 项目共享实例 |
| 上下文记忆 | 仅本次任务内 | 共享项目历史记忆 |
| 产出写入图谱 | 不写入 | 自动写入项目知识图谱 |
| 成员协作 | 仅创建者 | 项目成员可访问 |
| 审批 | 发回创建者自己的 HubAgent | 发送给项目负责人 HubAgent |

> 独立任务页复用项目内任务页的执行流组件，差异仅在 WorkAgent 实例和上下文持久化。

## 八、权限与数据归属

### 8.1 创建权限规则

**项目创建**：
- 所有 Member 和 Admin 均可创建
- 创建者自动成为项目负责人
- 系统同步初始化该项目的 WorkAgent 实例

**任务创建**：
- 仅项目负责人可以在项目内创建任务
- 普通成员不能自主创建任务
- 执行者选择范围：项目内成员 + 未在项目中的 Member（派发后自动加入）

**成员加入项目的唯一方式**：
- 被负责人派发任务后自动加入
- 不支持主动申请加入
- 不支持单独邀请（必须同时派发任务）

**自由对话**：
- Sidebar「新建对话」入口
- 与 HubAgent 自由聊天，完全独立，不属于任何项目
- 对话历史存入 HubAgent 个人记忆

### 8.2 数据归属矩阵

| 数据类型 | 归属 | 说明 |
|---------|------|------|
| HubAgent 对话历史 | 该 Member 个人 | 离职后可选择导出或删除 |
| 自由对话记录 | 该 Member 个人 | 同上 |
| 任务执行记录（过程）| 任务执行者 | 私有，不随项目转移 |
| 任务产出物 | 项目 | 写入 WorkAgent 项目记忆后归项目所有 |
| 知识图谱 | 项目 | 归项目所有，项目归档后由 IT 管理员管理 |
| 用户自建 Skill（私有）| 创建者个人 | 离职后随账号处理 |
| 用户自建 Skill（项目共享）| 项目 | 归项目所有 |
| Token 消耗记录 | 组织 | 由 IT 管理员在管理后台查看 |

### 8.3 特殊场景

**成员离职**：

| 数据 | 处理方式 |
|------|---------|
| HubAgent 个人记忆 | 账号停用后冻结，IT 管理员决定保留或删除 |
| 已写入项目的任务产出 | 保留在项目记忆中，不受影响 |
| 未完成的任务 | 负责人重新派发给其他成员 |
| 私有 Skill | 随账号冻结，IT 管理员可决定是否转移 |
| OrgAgent 的 HubAgent Summary | 已聚合统计数据保留，原始来源标注「已离职成员」|

**项目归档**：

| 数据 | 处理方式 |
|------|---------|
| WorkAgent 项目记忆 | 归档保留，只读，不可继续写入 |
| 知识图谱 | 归档保留，只读 |
| 项目内任务记录 | 归档保留 |
| WorkAgent 实例 | 停止运行，记忆冻结 |

### 9.2 关键实现要点

| 模块 | 实现要点 |
|------|---------|
| **全局布局** | 两栏结构：Sidebar（240px/56px）+ 主内容区，`flex-1 min-w-0` 弹性填充 |
