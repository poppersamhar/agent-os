# AgentOS 产品介绍（售前版）

> 面向热力/公用事业行业 · 业务决策者阅读

---

## 一、产品概述

**AgentOS 是企业级智能体操作系统**，为组织提供"数字员工"的招聘、管理、协作与进化能力。不同于传统的对话式 AI 工具，AgentOS 将大模型能力封装为可执行任务、可协作流转、可沉淀知识的智能体体系，让每个业务环节都有"懂业务、会执行、能协作"的数字员工参与。

在热力/公用事业行业，AgentOS 帮助企业将分散在各业务系统中的数据、流程和专家经验，转化为可复用、可编排、可度量的智能能力，实现从"人找信息"到"智能体主动服务业务"的转变。

## 二、核心能力

### 2.1 三层智能体架构，覆盖组织全层级

AgentOS 采用 **企业效能管理智能体 → 管理智能体 → 数字员工** 三层架构：

| 层级 | 角色 | 业务价值 |
|------|------|---------|
| **企业效能管理智能体（OrgAgent）** | 全局效能管理者 | 为管理层/员工提供全局数据看板、跨项目资源调度、组织级知识沉淀与经营分析 |
| **管理智能体（WorkAgent）** | 项目任务编排者 | 为每个项目提供任务拆解、方案规划、数字员工调度与结果汇总 |
| **数字员工（Sub-Agent）** | 原子任务执行者 | 按管理智能体编排的步骤，调用 Skill/Connector 完成数据采集、文档生成、分析计算等具体操作 |

**业务价值**：管理层看全局、员工有助手、任务能自闭环，真正实现"人机协同"而非"人机交替"。

### 2.2 项目化任务管理，业务过程可追踪

AgentOS 以"项目 → 任务 → 执行流"为主线组织工作：

- **项目**：对应一个完整业务目标（如"2024 供暖季客服知识库建设"）
- **任务**：项目内的具体工作项，由管理智能体（WorkAgent）驱动执行
- **执行流**：数字员工（Sub-Agent）按步骤自动执行，关键节点触发人工审批

所有任务过程留痕、产物自动归档、进度实时可视，彻底解决传统 AI 工具"问完即走、成果难沉淀"的问题。

### 2.3 三层上下文记忆，知识越用越聪明

AgentOS 具备任务级 → 项目级 → 全局级三层上下文记忆：

- **任务级**：单次任务的上下文，保证对话连续性
- **项目级**：同一项目下所有任务的产出自动汇总为项目知识文档
- **全局级**：跨项目的组织知识图谱，支持全局检索与关联分析

**业务价值**：新员工加入项目即可看到完整历史上下文；专家经验通过任务执行自动沉淀为组织资产，避免"人走知识丢"。

### 2.4 Skills & Connectors 生态，能力可复用可扩展

- **Skills（技能资产）**：将高频业务场景封装为可复用的执行模板，如"供暖投诉分析 Skill""能耗日报生成 Skill"，一次配置、全员复用
- **Connectors（连接器）**：通过 MCP 协议对接企业现有系统（ERP、CRM、SCADA、OA 等），也支持第三方数字员工的接入。智能体可直接查询业务数据、发起审批流程、回写操作结果

**业务价值**：不推翻现有 IT 建设，AgentOS 作为"智能中间层"，打通数据孤岛，让 AI 真正落到业务系统里。

### 2.5 全局知识库，组织资产一盘棋

AgentOS 将知识库从"项目附属功能"升级为"组织级基础设施"：

- **全局知识图谱**：可视化呈现全组织的项目、Agent、Skill、知识源之间的关联关系
- **多源知识接入**：企业知识库、项目沉淀、第三方数据库、MCP 服务统一纳管
- **知识随用随取**：员工在任何页面都可通过 OrgAgent 对话直接检索全局知识

**业务价值**：供热规章制度、应急预案、设备台账等组织资产真正"活"起来，打破项目壁垒，供全员随时调用。

### 2.6 审批与协作闭环，安全可控

数字员工在执行过程中遇到敏感操作（如费用审批、数据导出、对外发送）自动暂停，通过企业效能管理智能体（OrgAgent）向任务负责人推送审批卡片。审批通过后任务自动恢复执行。

**业务价值**：既享受了 AI 自动化的效率，又保留了人工审核的合规性，满足国企/公用事业单位的内控要求。

## 三、热力行业典型应用场景

### 场景一：管网智能巡检
- **痛点**：巡检记录分散在纸质表格和多个系统中，故障分析依赖老专家经验
- **方案**：WorkAgent 接收巡检计划 → 通过 Connector 调用 GIS 获取管网地图 → 调度视觉识别数字员工分析巡检照片 → 自动标记缺陷位置与等级 → 生成巡检报告并归档
- **收益**：故障响应时间缩短 40%，巡检记录 100% 数字化归档

### 场景二：供热调度优化
- **痛点**：调度决策依赖人工经验，气象变化与热负荷匹配响应滞后
- **方案**：基于气象数据与历史能耗，WorkAgent 自动编排气象分析、负荷预测、管网仿真等数字员工，生成 72 小时调度建议，人工确认关键节点后执行
- **收益**：调度决策响应从天级缩短至分钟级，附带热负荷预测与风险预警

### 场景三：经营分析与报告生成
- **痛点**：月度经营报告需多人跨系统取数、手工汇总、反复校对
- **方案**：管理智能体自动连接财务系统、生产系统、客服系统 → 按模板生成分析报告 → 关键指标异常自动标注
- **收益**：报告生成从 3 天缩短至 30 分钟，数据口径统一可追溯

### 场景四：知识管理与新人培训
- **痛点**：供热规章制度、应急预案、技术文档分散，新人上手周期长
- **方案**：历史文档自动入库 → 构建企业知识图谱 → 企业效能管理智能体（OrgAgent）随时问答 → 培训材料自动生成
- **收益**：新人培训周期缩短 50%，知识检索效率提升 80%

## 四、产品优势

| 维度 | 传统 AI 对话工具 | AgentOS |
|------|-----------------|---------|
| **工作组织** | 单轮对话，无过程管理 | 项目-任务-执行流闭环，过程可追溯 |
| **知识沉淀** | 对话历史难检索、难复用 | 三层记忆自动沉淀，全局知识图谱可视化 |
| **系统集成** | 无法对接企业系统 | MCP 协议对接 ERP/CRM/SCADA/OA，数字员工可外部接入 |
| **权限管控** | 全员共享同一权限 | Member/Admin 分级、任务级审批、数据隔离 |
| **能力扩展** | 依赖厂商更新 | Skills 自建 + Connectors 生态 + 第三方数字员工，持续进化 |
| **知识管理** | 无组织级知识视角 | 全局知识库独立管理，多源接入，跨项目共享 |

AgentOS 不是聊天助手，也不是传统 RPA，而是占据"通用对话"与"企业系统"之间一直缺失的**智能体编排与能力资产层**：

```
┌─────────────────────────────────────┐
│  通用 AI 助手（ChatGPT/文心一言等）   │  ← 能对话，不能执行，不能进业务系统
├─────────────────────────────────────┤
│  ★ AgentOS 智能体编排与能力资产层 ★  │  ← 任务编排、Skill 资产、全局知识库
├─────────────────────────────────────┤
│  企业现有系统（ERP/SCADA/OA/CRM）     │  ← 有数据，缺智能
└─────────────────────────────────────┘
```

| 对比维度 | 传统 AI 对话工具 | 传统 RPA | **AgentOS** |
|---------|-----------------|---------|------------|
| **工作组织** | 单轮对话，无过程管理 | 固定脚本流程 | **项目-任务-执行流闭环，过程可追溯** |
| **系统集成** | 无法对接企业系统 | API 直连，无智能 | **MCP 协议统一接入，数字员工可外部接入** |
| **知识沉淀** | 对话历史难检索、难复用 | — | **三层记忆自动沉淀，全局知识图谱可视化** |
| **能力扩展** | 依赖厂商更新 | 需重新编写脚本 | **Skills 自建 + Connectors 生态 + 第三方数字员工，持续进化** |
| **人机协作** | 人机交替 | 人机交替 | **人机协同：人决策审批，Agent 执行归档** |

## 五、部署与交付

- **部署方式**：支持私有化部署（本地服务器/私有云），数据不出域，满足公用事业单位信息安全要求
- **接入周期**：标准环境 1-2 周完成部署，Skills 配置与系统对接 2-4 周
- **使用方式**：Web 端直接访问，无需安装客户端，支持企业微信/钉钉单点登录
